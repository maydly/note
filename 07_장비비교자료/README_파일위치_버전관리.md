## 3. GitHub Pages 통합 규정

- 이 폴더는 나중에 `bucket0825.github.io/note/` 아래에 그대로 붙일 수 있는 정적 사이트 모듈로 관리한다.
- HTML 안에서는 내 컴퓨터 절대경로를 쓰지 않는다.
- 내부 이동은 같은 폴더 기준 상대경로만 쓴다.
- 이미지 등 로컬 assets는 `07_장비비교자료/assets/` 안에 둔다.
- 조명 이미지 assets는 `07_장비비교자료/assets/lighting/`를 기준으로 한다.
- 공통 디자인 파일은 `07_장비비교자료/assets/` 안에서 유지한다.
- 현재 `note` 사이트가 Design, Development, AI, Workflow 섹션 구조이므로, 통합 시에는 Design 쪽의 촬영 장비/조명 공부 자료로 연결하는 것을 우선 후보로 둔다.

## 4. 파일명 규정

- 최신 원본은 `07_장비비교자료/*_latest.html` 파일명을 고정한다.
- 사용자가 찾는 파일명은 매번 바꾸지 않는다.
- 버전 번호는 최신본 파일명 뒤에 붙이지 않고 변경기록과 `VERSION_MANIFEST.json`에 남긴다.
- 보존본이 필요할 때만 `07_장비비교자료/versions/YYYY-MM-DD/` 안에 복사본을 만든다.

## 5. 에이전트 작업 규정

- 새 장비를 추가할 때는 해당 `*_latest.html`, `CHANGELOG.md`, `VERSION_MANIFEST.json`을 같이 갱신한다.
- 제품 가격은 원화로 표시한다.
- 제품 이미지는 제품별로 다른 이미지를 쓴다.
- 제품 링크는 브랜드 메인이 아니라 해당 제품 페이지를 우선한다.
- 카메라/렌즈/필터/조명/악세사리/촬영지원장비를 한 페이지에 섞지 않는다.
- 같은급 비교가 필요한 경우 브랜드별 나열보다 비교군별 묶음을 먼저 만든다.
- 목차, 카메라, 렌즈, 필터, 조명, 악세사리, 촬영지원 상단 내비게이션은 모든 페이지에서 고정 노출한다.
