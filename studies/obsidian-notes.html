옵시디언 —
연결되는 노트 만들기

흩어진 메모·기록·계획을 링크로 엮어 "다시 찾아지는" 지식 그물로 만드는 노트 도구. 옵시디언이 정확히 뭔지, 핵심 개념, 왜 부담 없이 시도할 만한지, 시작하는 법과 주의점까지 — 처음 봐도 끝까지 읽히게 정리했다.

옵시디언 노트·기록 지식관리 마크다운

이 노트를 읽고 나면

사전 지식 없이도 "이게 나한테 맞나"를 판단할 수 있게.

  • ✓옵시디언이 "프로그램"이 아니라 "내 글 폴더 위에 얹는 도구"임을 안다.
  • ✓Vault·위키링크·백링크·그래프 같은 핵심 개념을 안다.
  • ✓왜 git·다른 편집기와 잘 어울리는지 안다.
  • ✓부담 없이 1주일 체험으로 시작하는 법과, 클라우드 동기화 주의점을 안다.
30초 결론 — 옮기는 수고가 사실상 없다(이미 마크다운 글이면 폴더만 열면 됨). "안 맞으면 그냥 안 쓰면 되는" 부담 없는 시도다. 단, 클라우드 동기화 폴더 안에서 쓸 땐 충돌 주의(아래 05).

01 — 옵시디언이 정확히 무엇인가

데이터베이스도, 특정 회사 클라우드도 아니다.

옵시디언은 내 컴퓨터 안의 글 파일 폴더 위에서 동작하는, "서로 연결된 노트를 보고 쓰는 프로그램"이다. 따로 데이터베이스를 두지도, 특정 회사 서버에 묶이지도 않는다. 폴더 안의 .md(마크다운) 파일을 읽어 링크·백링크·그래프·검색·태그를 얹어주는 "한 겹의 도구"일 뿐이다.

가장 중요한 성질 — 파일이 원본(진짜 본체)이다. 옵시디언을 지워도 글은 그대로 남고, 메모장·VS Code·다른 편집기로도 똑같이 열린다. 특정 앱에 갇히지 않는다.

02 — 핵심 개념 다섯

이것만 알면 화면이 읽힌다.

보관소

Vault (볼트)

옵시디언이 한 묶음으로 다루는 폴더 하나. 그 안의 글들이 서로 연결된다. "이 폴더가 내 노트 세계"라는 단위.

파일

.md (마크다운)

#, * 같은 간단한 기호로 제목·굵게·목록을 표시하는 쉬운 글쓰기 형식. 이름 끝이 .md인 그냥 글자 파일.

연결

위키링크 [[ ]]

대괄호 두 개 안에 문서 이름을 넣어 다른 글로 잇는 표기. [[작업기록]]처럼. 위키백과의 파란 링크와 같은 원리.

역방향

백링크

"이 글을 가리키는 다른 글 목록"을 자동으로 모아 보여주는 기능. 내가 잇지 않아도, 나를 가리킨 글이 보인다.

전체 지도

그래프 (Graph)

글을 점, 링크를 선으로 그린 관계 지도. 노트가 쌓일수록 "내 지식이 어떻게 연결돼 있나"를 한눈에 본다.

03 — 왜 부담 없이 시도할 만한가

갇히지 않고, 도구와 잘 어울린다.

  • ✓파일이 원본 — 앱에 종속되지 않는다. 옵시디언을 떠나도 글은 그대로.
  • ✓플레인 텍스트 — 꾸밈 정보 없는 순수 글자라, 어떤 프로그램으로도 열리고 오래간다.
  • ✓git과 잘 맞음 — 글자 그대로 저장이라 버전관리(언제 뭐가 바뀌었는지)와 궁합이 좋다. → git 노트
  • ✓연결망 — 흩어진 메모·기록이 백링크·그래프로 묶여, 한 글에서 관련 글로 따라갈 수 있다.
  • ✓옮기는 비용 0 — 이미 마크다운으로 쓰고 있다면 폴더만 열면 끝.

04 — 부담 없이 시작하기

실제 폴더에 바로 붙이지 말고, 체험부터.

1

연습 Vault 만들기

실제 작업 폴더 말고, 따로 빈 연습 폴더 하나를 Vault로 연다.

2

노트 몇 개 + 링크

글 두세 개를 쓰고 [[다른노트]]로 이어본다. 백링크가 잡히는 걸 확인.

3

그래프 열어보기

관계 지도가 어떻게 그려지는지 본다. 감을 잡는 단계.

4

1주일 체험

실제 기록 몇 개를 옮겨 일상에서 써본다. 손에 맞는지 본다.

5

괜찮으면 정식 연결

맞으면 실제 작업 폴더를 Vault로 연결. 안 맞으면 그냥 연습 폴더만 지우면 끝.

05 — 클라우드 동기화 주의

Drive·드롭박스 폴더 안에서 쓸 때.

작업 폴더가 구글 드라이브 같은 동기화 폴더 안에 있으면, 처음에 정리해 둘 게 있다. 동기화가 같은 파일을 동시에 건드리면 충돌 사본이 생기거나 설정이 꼬일 수 있다.

  • !충돌 사본 — 여러 기기에서 동시에 열면 "(1)" 붙은 사본이 생길 수 있다. 한 기기에서 닫고 동기화된 뒤 다른 기기에서 연다.
  • !.obsidian 설정 폴더 — Vault 안에 옵시디언 설정이 숨어 들어간다. 이게 동기화로 꼬이면 설정만 따로 관리하거나 동기화 제외를 고려.
  • !git과 겹칠 때 — 같은 폴더에 git까지 있으면 동기화 충돌이 git을 깨뜨릴 수 있다. 가능하면 동기화 폴더 밖에 두는 게 안전. → git×Drive 복구 노트

06 — 확장 4단계

처음부터 다 안 해도 된다. 천천히 올린다.

단계무엇을
1. 기본그냥 마크다운으로 글 쓰기. 메모장 대신 옵시디언으로.
2. 연결[[위키링크]]로 글을 잇고 백링크·그래프 활용.
3. 플러그인달력·칸반·표 같은 확장 기능을 필요한 것만 추가.
4. 자동화템플릿·정기노트 등으로 반복 작업을 줄임.
흔한 실수 — 처음부터 플러그인을 잔뜩 깔고 복잡하게 만들기. 1~2단계만으로도 충분히 쓸모 있다. 손에 익은 뒤 천천히 올린다.
