여러 페이지를 만들며 익힌 정적 사이트 폴더 구조와, 흔들리지 않는 CSS 레이아웃의 기본기. 이 아카이브를 만들며 정리한 패턴.

01 — Structure

파일 구조 잡기

한 페이지가 아니라 "사이트"가 되면, 폴더를 어떻게 나누느냐가 관건이 된다.

공용과 개별을 나눈다

Pattern

랜딩(첫 화면)과 하위 노트 페이지는 성격이 다르다. 그래서 CSS도 나눠 두면 안전하다. 랜딩만 손볼 때 공용 스타일을 건드려 하위 페이지가 통째로 깨지는 사고를 막을 수 있다.

landing.css

첫 화면(랜딩) 전용. 여기만 고치면 하위 페이지는 안전하다.

styles.css

하위 노트 페이지 공용. 모든 노트가 함께 쓰는 디자인.

studies/

개별 노트 HTML을 모으는 폴더. 템플릿을 복사해 시작.

템플릿을 둔다

Reuse

새 노트를 만들 때마다 빈 화면에서 시작하지 않는다. template.html 하나를 만들어 두고 복사해서 제목·내용만 바꾼다. 디자인이 저절로 통일되고, 만드는 속도도 빨라진다.

02 — Layout

흔들리지 않는 레이아웃

화면 크기가 달라져도 무너지지 않게 만드는 최소한의 도구.

기억할 도구

Tools
  • max-width + margin:auto — 본문 폭을 적당히 잡고 가운데 정렬. 큰 화면에서 글이 너무 늘어나지 않게.
  • CSS 변수(:root) — 색·여백을 한 곳에 모아 두면, 톤을 바꿀 때 한 줄만 고치면 된다.
  • Grid / Flex — 카드 나열은 grid, 좌우 정렬은 flex. 둘이면 거의 다 된다.
  • clamp() — 글자 크기를 화면에 따라 자동으로. 모바일·데스크톱 둘 다 자연스럽게.
  • @media — 좁은 화면에서 2열을 1열로 바꾸는 식의 반응형 한 줄.

밝은 톤을 기본으로

Tone

이 아카이브는 종이 같은 밝은 배경(베이지·아이보리)을 기본으로 한다. 어두운 색은 커버 밴드 같은 강조에만 쓰고, 본문은 읽기 편한 라이트 톤을 유지한다. 색 한두 개(accent)로 포인트만 주면 차분하면서도 심심하지 않다.

03 — Keep light

가볍게 유지하기

정적 사이트의 강점은 빠름. 그걸 지키는 습관.

무게 관리

Performance
  • 이미지는 적정 크기로 줄여서 넣는다. 원본 그대로 넣으면 페이지가 무거워진다.
  • 이미지를 HTML 안에 통째로 박아넣으면(base64) 파일이 수 MB로 부푼다 — 가능하면 파일로 분리.
  • 폰트는 필요한 굵기만 불러온다.
  • 외부 스크립트는 꼭 필요한 것만. 정적 사이트의 장점은 "가벼움"이다.
