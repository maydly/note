열어보지 않아도
아는 파일 정리법

작업이 쌓일수록 "그 파일 어디 갔지"가 늘어난다. 이름·폴더·버전만 잘 잡으면 검색하지 않고도 찾는다. 파일명 규칙, 날짜·버전 표기, 덮어쓰지 않는 버전관리, 업무ID 묶음까지 — 직접 쓰며 다듬은 체계를 정리했다.

파일명 규칙 버전관리 업무ID 정리 체계

이 노트를 읽고 나면

반년 뒤의 내가 헤매지 않게.

  • ✓파일명만 보고 내용·상태를 알 수 있게 짓는 법을 안다.
  • ✓버전 파일에 날짜·버전을 붙이는 표준 형식을 안다.
  • ✓덮어쓰지 않고 버전을 쌓아 롤백 길을 남기는 법을 안다.
  • ✓업무ID로 산출물을 묶어 추적하는 구조를 안다.
한 줄 요약 — 기준은 단 하나. 파일 목록을 보는 사람이 이름만으로 "이게 뭔지" 즉시 안다. 그게 안 되면 정리한 게 아니다.

01 — 이름만 보고 아는 파일

"열어봐야 아는" 게 아니라 "보면 아는" 이름.

대상 + 세부내용 + 유형을 모두 이름에 담는다. 정리·자료·리스트 같은 단독 단어는 금지 — 무엇에 대한 건지 알 수 없으니까. 수치·규격·목적이 있으면 이름에 넣는다.

나쁜 예 (열어봐야 앎)좋은 예 (이름만 봐도 앎)
리스트.html조명_브랜드비교표_추천리스트.html
보고서.html장비비교도구_작업보고서.html
플랜.md예약시스템개편_실행플랜.md

02 — 날짜·버전 표기

순서가 중요하다. 날짜를 앞에 두지 않는다.

버전 파일은 이름 → 날짜시간 → 버전 순으로 적는다. 24시간제, 구분자는 언더스코어.

형식 — (파일명)_YYMMDD_HHMM_v버전.확장자
예시 — 랜딩페이지_260630_0153_v1.1.html
  • 같은 분(分)에 여러 번 저장해도 버전번호(v1.1, v1.2)로 구분하니 초 단위는 안 넣는다.
  • 최종본(버전 폴더 밖)에는 날짜·버전을 붙이지 않는다. 항상 "지금의 최신"만 깔끔하게.

03 — 덮어쓰지 않는 버전관리

모든 수정은 새 버전으로. 돌아갈 길을 남긴다.

원칙

덮어쓰기 금지

기존 파일을 고쳐 저장하지 않는다. 새 버전 파일을 만든다. 이전 버전이 곧 롤백 기준점.

최종본 위치

폴더 밖에 꺼내둠

최신 최종본은 버전 폴더 바로 밖에 둔다. 버전들은 폴더 안에 쌓는다.

변경 성격버전 올림
오타·문구·작은 보완v1.0 → v1.1
기능 추가·구조 변경·방향 전환v1.0 → v2.0
작업폴더/
├─ 랜딩페이지.html              ← 최종본(밖, 날짜·버전 없음)
└─ 랜딩페이지/                  ← 버전 폴더
   ├─ 랜딩페이지_260603_0153_v1.0.html
   └─ 랜딩페이지_260603_0210_v1.1.html

04 — 같은 파일의 발전 버전

이름이 바뀌며 진화했어도 한 흐름으로 본다.

같은 목적의 결과물이 이름·구조를 바꾸며 여러 단계로 발전했다면, 모든 버전을 한 버전 폴더에 모으고 순차 번호로 잇는다. 흩어 두면 어느 게 최신인지 알 수 없다. 폴더명은 최종 결과물 이름으로 통일하고, 버전 설명을 번호 앞에 남긴다.

스튜디오가이드/
├─ 가이드_260530_0111_v0.1.html        ← 초기 초안
├─ 가이드_안내페이지_260530_0111_v0.2.html ← 명칭 바뀐 시기
├─ 가이드_복사본_260530_0111_v0.3.html   ← 수정 복사본
└─ 가이드_인덱스_260530_0111_v0.4.html   ← 최종 배포본
하지 말 것 — 버전마다 별도 폴더 만들기 / 전부 v0.1로 표기(순서 모름) / 최종본 없이 버전 폴더만 두기.

05 — 업무 단위로 묶기 (업무ID)

파일 하나가 아니라 "어느 업무에서 나온 것"으로 추적.

모든 산출물에 업무ID를 붙여 폴더로 묶는다. 형식은 대상-분야-번호_업무명. 그러면 파일 하나만 봐도 "어느 프로젝트의 어떤 업무"인지 즉시 안다.

예시 — maydly-운영-006_학습아카이브콘텐츠정비 · 장비-조사-001_조명비교
1

업무ID 폴더 생성

업무를 시작하면 먼저 ID 폴더를 만든다.

2

업무카드

그 안에 개요·결정·상태를 적은 카드를 둔다.

3

계획 → 산출물 → 보고서

정해진 하위 폴더에 단계별로 쌓는다.

4

중단·폐기도 보존

버린 것도 지우지 않고 보존 폴더에 남긴다.

📚 더 깊이 — 강의용 상세

위가 요약이라면, 여기는 자세히 배우는 곳. 탭을 눌러 펼쳐 보세요.

이름·버전은 "정리"가 아니라 "미래의 나를 위한 안내판"이다

파일명을 짓는 일은 예뻐 보이려는 게 아니다. 반년 뒤의 내가 아무 기억 없이 폴더를 열었을 때, 헤매지 않게 미리 남겨두는 안내판이다. 촬영본 수백 장, 시안 여러 벌이 쌓이는 작업일수록 "그 파일 어디 갔지"가 곧 시간 손실이 된다. 이름만으로 내용·상태를 알 수 있으면 검색조차 필요 없다.

버전관리도 같은 마음에서 나온다. 사진을 보정할 때 원본을 지우지 않고 사본에서 작업하듯, 파일도 이전 것을 덮어쓰지 않고 새 버전을 쌓는다. 그러면 방향을 잘못 잡았을 때 언제든 이전 지점으로 돌아갈 수 있다. 덮어쓴 순간, 돌아갈 길은 사라진다. 버전 폴더는 그 "돌아갈 길"을 통째로 보관하는 서랍이다.

핵심은 규칙이 복잡해서가 아니라, 기준이 딱 하나라서 지켜진다는 점이다 — "목록을 보는 사람이 이름만으로 이게 뭔지 즉시 아는가." 이 한 문장이 통과되면 정리된 것이고, 안 되면 아무리 폴더를 나눠도 정리가 아니다.

시안 파일 하나를 이름 짓고 버전 올려보기

  • 1) 대상+세부+유형으로 짓기. 시안.html 대신 가을화보_예약안내페이지_시안.html처럼. 열지 않아도 무엇인지 보인다.
  • 2) 첫 버전 저장. 버전 폴더를 만들고 그 안에 가을화보_예약안내페이지_시안_260711_1430_v1.0.html. 이름 → 날짜시간 → 버전 순서를 지킨다.
  • 3) 오타 하나 고칠 때. 원본을 덮지 않고 …_260711_1615_v1.1.html로 새로 저장. 작은 수정이니 소수점만 올린다.
  • 4) 구성을 크게 바꿀 때. 방향 전환이면 …_v2.0.html. 이전 v1 계열은 그대로 서랍에 남는다.
  • 5) 최종본 꺼내두기. 지금의 최신 한 벌만 버전 폴더 밖에 가을화보_예약안내페이지_시안.html(날짜·버전 없이). "이게 지금 것"이 한눈에 보인다.

보다시피 파일은 늘어나도 헷갈리지 않는다. 밖에 있는 것이 최신, 안에 쌓인 것이 지나온 길이다.

이런 데서 정리가 무너진다

  • ① 덮어쓰기. "조금만 고치는 건데" 하고 원본 위에 저장 → 돌아갈 지점이 사라진다. 잘못 고친 걸 깨달아도 복구 불가. → 항상 새 버전으로.
  • ② 두루뭉술한 이름. 최종.html, 진짜최종.html, 찐최종.html → 뭐가 진짜인지 아무도 모른다. → 대상+세부+유형과 버전번호로.
  • ③ 날짜를 앞에 두기. 260711_시안.html처럼 날짜부터 쓰면 목록이 날짜순으로만 정렬돼 무슨 파일인지 안 보인다. → 이름을 앞, 날짜·버전은 뒤에.
  • ④ 버전마다 폴더 따로. 시안1/, 시안수정/, 시안진짜/로 흩어두면 흐름을 잃는다. → 한 버전 폴더에 순차 번호로 모은다.
  • ⑤ 최종본이 없음. 버전 파일만 잔뜩 있고 밖에 최종본이 없으면, 매번 날짜를 비교해 "지금 것"을 찾아야 한다. → 최신 한 벌은 늘 폴더 밖에.

지금 내 파일로 해보기 (5분 연습)

바탕화면이나 작업 폴더에서 이름이 애매한 파일 하나를 골라, 아래 네 가지를 적용해 보세요.

  • 이름 다시 짓기 — 그 파일을 대상 + 세부내용 + 유형으로 바꿔 적어보기. (예: 겨울프로필_촬영준비물_안내.md)
  • 버전 번호 붙이기 — 지금 버전을 _YYMMDD_HHMM_v1.0 형식으로 만들어보기.
  • 다음 수정 상상하기 — 오타 수정이면 몇으로, 구조 변경이면 몇으로 올릴지 정해보기. (v1.1 / v2.0)
  • 최종본 자리 정하기 — 버전 폴더를 만들고, 최신 한 벌을 폴더 밖에 날짜·버전 없이 꺼내두기.

이 네 가지가 손에 붙으면, 파일이 아무리 쌓여도 "어디 뒀더라"가 사라집니다.

06 — 정리 점검

파일을 만들고 나서 빠르게.

  • ✓이름만 보고 내용·상태를 알 수 있는가
  • ✓버전 파일에 _YYMMDD_HHMM_v버전 형식을 지켰는가
  • ✓기존 파일을 덮어쓰지 않고 새 버전으로 만들었는가
  • ✓최종본이 버전 폴더 밖에 깔끔히 있는가
