서버 없이, 비용 없이, GitHub Pages로 정적 사이트를 띄우는 흐름. 이 아카이브 자체가 그렇게 배포되어 있다.

01 — What

GitHub Pages가 뭔가

저장소에 올린 HTML/CSS를 그대로 웹 주소로 보여주는 무료 호스팅.

한 줄 정의

Concept

GitHub에 코드를 올려두면, GitHub이 그 폴더를 그대로 웹사이트로 공개해 준다. 데이터베이스나 서버 프로그램이 없는 "정적" 사이트 — HTML, CSS, 이미지, 자바스크립트만으로 된 사이트에 딱 맞는다. 학습 아카이브, 포트폴리오, 안내 페이지 같은 용도에 이상적이다.

공개 주소 형태: https://사용자명.github.io/저장소명/
이 아카이브는 bucket0825.github.io/note/ 로 열린다.

02 — How

배포 흐름

처음 한 번만 설정하면, 그다음부터는 push만으로 갱신된다.

최초 설정

Setup
  1. GitHub에 저장소를 만든다.
  2. HTML·CSS가 든 폴더를 커밋하고 push 한다.
  3. 저장소 Settings → Pages로 간다.
  4. "Deploy from a branch" 선택, Branch는 main, 폴더는 / (root).
  5. 1~2분 뒤 공개 주소로 접속된다.

그 후의 갱신

Update

한 번 설정해두면 그다음은 단순하다. 파일을 고치고 → 커밋하고 → push하면, 1~2분 뒤 라이브 사이트에 자동 반영된다. 별도 배포 버튼이 없다. push = 배포라고 생각하면 된다.

그래서 "아직 안 보여주고 싶은 것"은 push하지 않는다. push하는 순간 공개된다는 걸 기억할 것.

03 — Watch out

걸리기 쉬운 곳

정적 사이트 특유의 함정 몇 가지.

주의 목록

Caution
  • 경로는 상대경로로. 폴더를 옮기면 링크가 깨지므로 구조를 함부로 안 바꾼다.
  • 이미지·CSS 파일명 대소문자를 정확히. 내 PC에선 되는데 공개 주소에선 깨지는 단골 원인.
  • 공개 사이트엔 내부 정보(메모·연락처·금액·키)를 절대 올리지 않는다.
  • 반영이 느리면 브라우저 캐시 탓일 때가 많다. 새로고침(강력 새로고침)으로 확인.
  • 커스텀 도메인을 붙이려면 Pages 설정에서 도메인 + DNS 설정이 추가로 필요하다.
